Build a full-stack AI Emotion-Based Music Recommendation System similar to Spotify.

Frontend:
React with TailwindCSS dark dark-mode UI like Spotify.

Backend:
Python FastAPI.

Music source:
Use YouTube Music search using the ytmusicapi Python library.

The music must play directly inside the website using an embedded YouTube player or audio player.

Main features:

- Search songs
- Play songs inside the website
- Pause songs
- Skip songs
- Like songs
- Create playlists
- Show recently played songs
- View listening history
- Save history in database

Recommendation logic:

1. When a user searches for a song, detect the emotion of the song (happy, sad, angry, calm, excited) using simple keyword or genre mapping.

2. If the user listens to the full song (90% or more), recommend similar songs from the same genre, artist, and emotion.

3. If the user skips the song (less than 20%), avoid similar songs and recommend songs from a different emotion category.

4. If the user listens halfway (30–60%) then skips, recommend songs with the same emotion but better similarity.

User behavior tracking:

Track:
- play_time
- song_duration
- play_percentage

Formula:
play_percentage = play_time / duration

Interpretation:
0-20% = skip
30-60% = partial like
90-100% = full like

Backend endpoints:

POST /search
Search songs using YouTube Music

POST /play
Start playback

POST /progress
Update play percentage

POST /skip
Mark song skipped

POST /recommend
Return recommended songs

POST /playlist/create
Create playlist

POST /playlist/add
Add song to playlist

GET /history
Return last 20 played songs
